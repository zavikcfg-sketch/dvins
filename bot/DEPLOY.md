# 🚀 Деплой на BotHost: сайт + бот на Dvin.bothost.tech

Единый Node.js-сервер раздаёт сайт, отдаёт API и запускает Telegram-бота.
Бот и сайт работают с **общими данными** — изменил в боте, видно на сайте.

---

## Структура, которую нужно загрузить на BotHost

```
(корень проекта на BotHost)
 ├── server.js            ← главный файл (из папки bot/)
 ├── package.json         ← из папки bot/
 ├── defaults/            ← из папки bot/  (стартовые данные)
 │    ├── rooms.json
 │    ├── gallery.json
 │    └── contacts.json
 └── public/              ← СОБРАННЫЙ САЙТ (см. шаг 2)
      ├── index.html
      └── images/
           ├── hero.jpg
           ├── guest-house.jpg
           ├── pool.jpg
           ├── room.jpg
           └── beach.jpg
```

> Папка `data/` создастся автоматически при первом запуске —
> туда сервер пишет актуальные номера/галерею/контакты.

## Шаг 1. Собрать сайт

На своём компьютере в корне проекта:

```bash
npm install
npm run build
```

Появится файл `dist/index.html`.

## Шаг 2. Подготовить папку `public`

Создайте папку `public` рядом с `server.js` и положите туда:

1. `dist/index.html`  →  `public/index.html`
2. всю папку `public/images/` из проекта  →  `public/images/`

> В этом проекте картинки уже лежат в `public/images/`.
> Итог: `public/index.html` + `public/images/*.jpg`.

## Шаг 3. Загрузить на BotHost

Залейте на BotHost содержимое папки `bot/` (server.js, package.json,
defaults/) **плюс** подготовленную папку `public/`.

Если деплой через Git — закоммитьте эти файлы и укажите рабочую папку.

## Шаг 4. Переменные окружения (в панели BotHost)

| Переменная   | Значение                                  | Обязательно |
|--------------|-------------------------------------------|-------------|
| `BOT_TOKEN`  | токен от @BotFather                        | да          |
| `ADMIN_ID`   | ваш Telegram ID (от @userinfobot)          | желательно  |
| `ADMIN_PASS` | пароль для веб-админки `#/admin`           | желательно  |

> `PORT` BotHost задаёт сам — не трогайте.
> Если `ADMIN_PASS` не задать, по умолчанию пароль `admin`.

## Шаг 5. Команды

- Установка: `npm install`
- Запуск: `npm start`  (или `node server.js`)
- Node.js: **18+**

## Шаг 6. Проверка

В логах BotHost должно появиться:

```
🌐 Сайт + API запущены на порту XXXX
🤖 Telegram-бот запущен.
```

Открой **https://Dvin.bothost.tech** — сайт работает.
Напиши боту `/start` — отвечает меню.
Сделай `/busy r1` — на сайте номер станет «Занят» (обновляется в течение ~8 сек).

## Как управлять

**Через бота** (Telegram):
- `/list`, `/add`, `/free id`, `/busy id`, `/price id сумма`, `/del id`

**Через веб-админку**: открой `https://Dvin.bothost.tech/#/admin`,
введи `ADMIN_PASS`. Там вкладки: Номера / Галерея / Контакты.

Обе панели меняют одни и те же данные на сервере.

## Частые вопросы

**Сайт открывается, но без картинок.**
Проверь, что папка `public/images/` загружена на сервер.

**Изменения из бота не видно на сайте.**
Сайт опрашивает сервер каждые ~8 сек. Обнови страницу. Убедись, что
сайт открыт именно с домена BotHost (а не локальный файл).

**Бот не отвечает.**
Проверь `BOT_TOKEN` и что в логах есть «Telegram-бот запущен».
Если у тебя ещё где-то запущен этот же бот — Telegram разрешает
только один polling, останови второй экземпляр.
