# 🏡 Сайт + Telegram-бот гостевого дома

## ⚠️ ОЧЕНЬ ВАЖНО: куда загружать файлы

BotHost запускает приложение из **корневой папки** `/app/`. Это значит, что файлы должны лежать так:

```
/app/http-wrapper.js   ✅ ПРАВИЛЬНО
/app/server.js
/app/package.json
/app/public/index.html
```

а **НЕ так:**

```
/app/bot/http-wrapper.js   ❌ НЕ РАБОТАЕТ (как у вас сейчас)
/app/bot/server.js
```

👉 **Нужно загружать СОДЕРЖИМОЕ папки `bot/`, а не саму папку `bot/`.**

---

## 📋 Что должно оказаться в корне `/app/` на BotHost

| Файл / папка       | Назначение                                 |
| ------------------ | ------------------------------------------ |
| `http-wrapper.js`  | Точка входа (BotHost запускает этот файл)  |
| `index.js`         | Запасная точка входа                       |
| `server.js`        | Главный код: сервер + бот + API            |
| `package.json`     | Зависимости                                |
| `public/`          | Папка с собранным сайтом                   |

После запуска появится `data.json` — это база номеров.

## 🚀 ПОШАГОВАЯ ИНСТРУКЦИЯ ДЕПЛОЯ

### Шаг 1. Получить токен и Telegram ID

1. В Telegram → **@BotFather** → `/newbot` → скопируйте токен
2. В Telegram → **@userinfobot** → `/start` → скопируйте `Id` (число)

### Шаг 2. Собрать сайт на компьютере

В **корне проекта** (там, где `package.json` и `src/`):

```bash
node scripts/build-for-bothost.js
```

После этого появится `bot/public/index.html` и `bot/public/images/`.

### Шаг 3. Залить файлы на BotHost ❗ ВНИМАТЕЛЬНО

1. Зайдите на ваш бот в bot-hosting.net → вкладка **Files**
2. **УДАЛИТЕ ВСЕ ФАЙЛЫ И ПАПКИ**, которые там есть (включая `bot/`, если она появилась после прошлой загрузки!).
3. **Откройте папку `bot/` на компьютере**.
4. **Выделите ВСЁ внутри неё** (Ctrl+A): `http-wrapper.js`, `index.js`, `server.js`, `package.json`, `public/`, `README.md`.
5. **Перетащите выделенное** в окно Files на BotHost.

❗ Если случайно перетащите саму папку `bot/` — внутри хоста получится `/app/bot/...` и снова будет ошибка. Перетаскивайте **содержимое** папки, а не папку.

После загрузки в Files должно быть видно:

```
http-wrapper.js
index.js
server.js
package.json
README.md
public/   (раскрывается, внутри index.html и images/)
```

### Шаг 4. Установить зависимости

В панели BotHost откройте **Console** (или **Terminal**) и выполните:

```bash
npm install
```

Дождитесь сообщения `Зависимости установлены ✓`. Если такой кнопки нет — ищите в панели **Install dependencies** или **NPM Install**.

### Шаг 5. Задать переменные окружения

В разделе **Environment Variables** добавьте:

| Имя          | Пример значения                  |
| ------------ | -------------------------------- |
| `BOT_TOKEN`  | `1234567890:AAH-ваш_токен`        |
| `ADMIN_ID`   | `123456789`                       |
| `ADMIN_PASS` | `мой_пароль`                      |
| `SITE_URL`   | `https://Dvin.bothost.tech`       |

> `PORT` BotHost задаёт сам — НЕ трогайте.

### Шаг 6. Запустить

Нажмите **Start** (или **Restart**, если уже запущено).

В **Logs** должно появиться:

```
🌐 HTTP сервер: http://localhost:XXXX
📁 Статика:    /app/public
💾 Данные:     /app/data.json
🤖 Telegram-бот запущен.
👤 Админ ID: 123456789
```

### Шаг 7. Проверить

- 🌐 Откройте в браузере: **https://Dvin.bothost.tech**
- 🤖 Напишите боту `/start` в Telegram

✅ Готово!

## ❓ Решение типичных ошибок

### `Cannot find module '/app/http-wrapper.js'`
Файлы лежат в `/app/bot/`, а не в `/app/`. Удалите всё в Files и перезалейте **содержимое** папки `bot/`, не саму папку (см. Шаг 3).

### `Cannot find module 'node-telegram-bot-api'` / `'express'`
Не выполнен `npm install`. Откройте Console на BotHost и запустите `npm install` (Шаг 4).

### Сайт открывается, но в админке кнопки не работают
Это нормально — нужно убедиться, что API доступен. Откройте `https://Dvin.bothost.tech/api/health` в браузере — должно вернуть `{"ok":true}`.

### «Сайт ещё не собран»
Папка `public/` не загружена или пустая. Выполните Шаг 2 заново и перезалейте.

### Бот пишет «Доступ запрещён»
`ADMIN_ID` не совпадает с вашим Telegram ID. Перепроверьте у `@userinfobot`.

## 🎮 Что умеет бот

- 📋 **Список номеров** — кнопочный список со статусом 🟢/🔴
- ➕ **Добавление** — пошагово (название → цена → вместимость → описание)
- 🔄 **Свободен/Занят** — одна кнопка
- 💰 **Изменение цены** — одной командой
- 🗑 **Удаление**
- 🌐 **Кнопка «Открыть сайт»** — ведёт на ваш домен

Команды: `/start`, `/list`, `/add`, `/help`, `/cancel`.

---

## 🔄 Как обновить сайт после правок

1. На компьютере: `node scripts/build-for-bothost.js`
2. На BotHost в Files: удалите старую папку `public/`
3. Перетащите новую `public/` из `bot/`
4. Нажмите **Restart**

База `data.json` не теряется — это отдельный файл.
