# 🤖 Запуск Telegram-бота на BotHost

Пошаговая инструкция, как развернуть бота управления номерами.

---

## Шаг 1. Создать бота в Telegram

1. Откройте Telegram, найдите **@BotFather**.
2. Отправьте команду `/newbot`.
3. Придумайте имя и username бота.
4. BotFather пришлёт **токен** вида `1234567890:AAH...` — сохраните его.

## Шаг 2. Узнать свой Telegram ID

1. Найдите в Telegram бота **@userinfobot** и напишите ему `/start`.
2. Он пришлёт ваш числовой **ID** (например `123456789`).
   Это нужно, чтобы управлять ботом могли только вы.

## Шаг 3. Загрузить файлы на BotHost

Залейте на хостинг содержимое папки `bot/`:

```
bot/
 ├── bot.js
 ├── package.json
 ├── rooms.json
 └── README.md
```

> Если на BotHost есть подключение через Git — запушьте репозиторий и
> укажите рабочую директорию `bot`.
> Если загрузка файлами/архивом — загрузите эти файлы.

## Шаг 4. Указать переменные окружения

В панели BotHost (раздел **Environment / Переменные окружения**) добавьте:

| Переменная   | Значение                          |
|--------------|-----------------------------------|
| `BOT_TOKEN`  | токен от BotFather                 |
| `ADMIN_ID`   | ваш Telegram ID                   |

> На некоторых тарифах токен называется `TOKEN` — бот понимает оба имени.
> Переменную `PORT` BotHost задаёт сам, трогать не нужно.

## Шаг 5. Команды установки и запуска

В настройках процесса BotHost укажите:

- **Команда установки:** `npm install`
- **Команда запуска:** `npm start`
  (или `node bot.js`)

Версия Node.js — **18 или выше**.

## Шаг 6. Проверка

1. Запустите процесс на BotHost — в логах должно появиться:
   ```
   🤖 Бот запущен. Ожидаю сообщения...
   ```
2. Напишите боту в Telegram `/start` — он ответит меню.

---

## 📋 Команды бота

| Команда                  | Действие                          |
|--------------------------|-----------------------------------|
| `/start`                 | показать меню                     |
| `/list`                  | список всех номеров с ID          |
| `/add`                   | добавить номер (по шагам)         |
| `/free <id>`             | пометить номер свободным          |
| `/busy <id>`             | пометить номер занятым            |
| `/price <id> <сумма>`    | изменить цену                     |
| `/del <id>`              | удалить номер                     |

ID номера берётся из вывода команды `/list`.

---

## ⚠️ Про синхронизацию с сайтом

Сейчас бот хранит данные в `rooms.json` **на сервере**, а сайт — в
**браузере (localStorage)**. Это разные хранилища.

Чтобы бот реально менял то, что видят гости на сайте, нужен общий
бэкенд (API). Если нужно — попроси добавить мини-сервер, который будет
отдавать `rooms.json`, а сайт настроить на чтение оттуда.
