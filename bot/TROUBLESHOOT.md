# 🩺 Сайт/бот не работают — что проверить

Сервер уже стартовал (`🌐 Сайт + API запущены`), значит код рабочий.
Осталось два момента: установка модуля бота и доступ к сайту по домену.

---

## 1. Бот: установить node-telegram-bot-api

Лог сказал: модуль не установлен. Я добавил **автоустановку при старте**,
но надёжнее установить заранее. На BotHost задай команду установки:

```
cd bot && npm install
```

или в настройках укажи **рабочую папку (Working dir) = `bot`** и
команду установки `npm install`.

Проверь, что задан **`BOT_TOKEN`** в переменных окружения
(в /health он покажет "set" или "missing", см. ниже).

После перезапуска в логах ждём:
```
🤖 Telegram-бот запущен.
```

## 2. Сайт: проверь доступность

Открой в браузере:

```
https://Dvin.bothost.tech/health
```

- Если открылся JSON `{"ok":true,"port":...,"botToken":"set/missing",...}`
  → **сервер виден из интернета.** Значит сайт тоже работает —
  открой https://Dvin.bothost.tech (обнови с Ctrl+F5).

- Если `/health` НЕ открывается (таймаут/502/404 от платформы)
  → проблема в портах/привязке домена на стороне BotHost. Смотри пункт 3.

---

## 3. Порт и привязка домена

В логе был порт **3000** — это значение по умолчанию, т.е. BotHost
**не передал свою переменную PORT**. Многие платформы требуют, чтобы
приложение слушало именно их порт.

Что сделать:
- В настройках BotHost найди, какой порт он ожидает (часто это
  переменная `PORT`, иногда поле «Port» в панели).
- Убедись, что тип сервиса — **Web / HTTP** (а не «worker/background»),
  иначе домен не привязывается.
- Если можно задать порт вручную — поставь переменную окружения
  `PORT` = тот порт, что слушает платформа (например 8080), либо
  открой в панели порт 3000.

Сервер уже слушает на `0.0.0.0` (все интерфейсы) — это правильно для
контейнеров, отдельно настраивать не нужно.

---

## 4. Если сайт открывается, но «пустой»/без картинок

Проверь, что папка `bot/public` залита на сервер и содержит:
```
bot/public/index.html
bot/public/images/*.jpg
```
Собери её командой (на своём ПК, в корне проекта):
```
npm run build
node bot/prepare-public.js
```
В логе при старте сервер печатает: `📁 Папка сайта: ...` — там должен
быть путь, где реально лежит index.html.
