# 📤 Как загрузить сайт на BotHost (финальный шаг)

Из логов видно ДВЕ вещи:

1. `index.html НЕ НАЙДЕН` → сайт не загружен в `/app/bot/public/`
2. `💾 Файл данных: /app/bot/data.json` → на хосте лежит **СТАРАЯ** версия
   server.js (у новой версии данные в папке `data/`, а не `data.json`)

Значит нужно: **залить новую версию файлов + собранный сайт.**

---

## Шаг 1 — собрать сайт (на своём компьютере)

В корне проекта по очереди:

```bash
npm install
npm run build
node bot/prepare-public.js
```

После этого появится папка `bot/public/` с файлами:
```
bot/public/index.html
bot/public/images/hero.jpg
bot/public/images/guest-house.jpg
bot/public/images/pool.jpg
bot/public/images/room.jpg
bot/public/images/beach.jpg
```

## Шаг 2 — залить на BotHost

Залей содержимое папки `bot/` так, чтобы на хосте было ИМЕННО так:

```
/app/bot/server.js          ← НОВАЯ версия (важно перезаписать старую!)
/app/bot/package.json
/app/bot/defaults/rooms.json
/app/bot/defaults/gallery.json
/app/bot/defaults/contacts.json
/app/bot/public/index.html  ← из шага 1
/app/bot/public/images/...   ← из шага 1
```

> ⚠️ Обязательно ПЕРЕЗАПИШИ старый `/app/bot/server.js` новым.
> Старый писал «data.json» — если останется, ничего не изменится.

### Если заливаешь через Console (терминал BotHost)

Самый быстрый способ проверить, что файлы на месте:

```bash
cd /app/bot
ls -la               # должны быть server.js, public/, defaults/
ls -la public        # должен быть index.html и папка images
cat public/index.html | head -c 100   # должен вывести начало HTML
```

Если `public/index.html` нет — значит он не загрузился, повтори Шаг 2.

## Шаг 3 — установить зависимости и перезапустить

```bash
cd /app/bot
npm install
```

Затем перезапусти сервис в панели BotHost.

## Шаг 4 — проверить

В логах должно быть (БЕЗ «НЕ НАЙДЕН»):
```
📁 Папка сайта: /app/bot/public
🌐 Сайт + API запущены на 0.0.0.0:3000
🤖 Telegram-бот запущен (кнопки + фото).
```

Открой:
- https://dvin.bothost.tech         → сайт
- https://dvin.bothost.tech/health  → JSON со статусом

---

## Про фото в боте

Ошибка `sendPhoto error: failed to get HTTP URL content` была из-за того,
что картинок не было на хосте. После загрузки `bot/public/images/`
бот будет брать фото из локальных файлов — всё заработает.

Если фото всё равно не грузится — бот просто покажет карточку текстом,
это не ломает работу.

- Запуск:    `cd bot && npm start`     (или `node bot/server.js`)
